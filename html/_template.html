_firstDay Number 0 시작 요일 설정 (0~6, weekdays 옵션의 배열 기준)
_multiple Number 1 여러 달 표시

지정 날짜의 형식은 항상 _format과 같아야 합니다.
onSelect string/function 캘린더 화면에서 날짜 선택 후 실행되는 콜백입니다.

파일 업로드mm_form-file

* data-file의 옵션으로 '_default': '기본값' 설정할 수 있습니다.
* input에 accept로 확장자를 지정할 수 있습니다.
* data-file의 옵션으로 '_fileName'(파일명)과 '_fileSize'(파일 용량)를 지정할 수 있습니다.

이미지 업로드mm_form-image

# 초기화 트리거
* mm.event.dispatch([file], 'update');
* 페이지에 처음 접속했을 때 실행됩니다.
* 그 외 초기화가 필요하거나 별도로 [data-file] 요소를 생성할 경우 실행합니다.
* mm.form.update(대상 또는 부모)으로 사용할 수 있습니다.

# 값 전달
* mm.data.get([file], 'data-file').file: [file]의 전달 값(type object / File)입니다.
* [file].file는 선택 취소했을 때 값이 사라지는 이슈가 있어 mm.data.get([file], 'data-file').file을 form data로 변환하여 사용합니다.
* mm.data.get([file], 'data-file').file을 가지고 있는 요소에는 __image-on 클래스가 추가됩니다.
* mm.data.get([file], 'data-file').file에는 [file].file와 FileReader로 로드한 result: data...base64, width, height, orientation 값을 추가로 포함하고 있습니다.
* mm.data.get([file], 'data-file').file.result는 원본, mm.data.get([file], 'data-file').file.view.result는 미리보기 이미지와 같은 비율로 변환된 이미지 데이터가 저장됩니다.
* 파일 선택 전 미리보기(data-file: { _default })가 있을 때에는 name 값으로 파일명이 아닌 경로가 들어가 있고, type 값이 없습니다.
* 미리보기를 삭제할 경우 값은 {}입니다.

폼 테이블mm_formbox, mm_formbox-caption, .mm_formbox-unit / mm_form

# mm_form
* mm_form은 단독사용이 가능합니다.
* 첫번 째 tr은 th, td width의 기준을 잡기 위한 tr이므로 항상 넣어줍니다. (visibility: hidden)
* th의 width는 기본 200px이며, 팝업 안의 폼테이블 th의 width값은 166px입니다.
* td의 width는 colspan으로 조절합니다.

# mm_formbox
* mm_form의 상/하단에 관련 영역을 추가할 경우 mm_formbox로 감싸줍니다.

검색mm_search

* 첫번 째 tr은 th, td width의 기준을 잡기 위한 tr이므로 항상 넣어줍니다. (visibility: hidden)
* th의 width는 105px 고정입니다.
* td의 width는 colspan으로 조절합니다.

내용없음mm_form-none

* mm_form 안 table 영역을 삭제하고 mm_form-none을 넣어줍니다.
* 상태 및 조건에 따라 문구를 변경합니다.
  기본 : 등록된 내용이 없습니다.
  조건 예시 : 자동회수인 경우에만 회수지 정보가 노출됩니다.

오류메시지

* 폼요소의 오류메시지는 스크립트 mm.form.alert(대상 input, '알림 메시지')를 호출하여 사용할 수 있습니다.

업데이트

* 폼 요소의 생성, 초기화, 변경 등 변화가 발생하면 업데이트가 필요합니다.
* mm.form.update();로 업데이트 합니다.

* 파라미터가 빈 값이면 html 전체 폼 요소를 검색하여 업데이트 합니다.
* 파라미터가 폼 요소일 경우 선택한 폼 요소만 업데이트 합니다.(이 때 아래 샘플처럼 변경과 업데이트를 동시에 적용할 수 있습니다.)
* 파라미터가 폼 요소의 컨테이너 요소일 경우 컨테이너 내 모든 폼 요소를 업데이트 합니다.

기본 테이블mm_table

# 기본 테이블
* mm_table은 단독으로 사용이 가능합니다.
* mm_table에 data-table 속성을 기본으로 적용하고 그 외 필요한 속성을 추가합니다.
* 테이블의 너비값은 자동으로 적용되고,
  col로 각 영역의 너비값을 지정할 수도 있습니다. (너비를 지정 시 테이블 기능은 적용되지 않습니다.)
* thead 요소는 상단 고정으로 분할됩니다.
* ajax로 변경 시 mm_table 요소 내 mm_scroll 요소까지 포함하여 변경 후 update를 실행합니다.

# mm_table
* mm.table.update(selector) :: 테이블 분할 변경 (페이지 로드 시 update)
* mm.table.remove(selector) :: 테이블 삭제
* mm.table.destroy(selector) :: 테이블 해제
* mm.table.resize(selector) :: 테이블 리사이즈

기능 테이블__table-fixed, _height

# 높이 제한 (max-height)
* data-table="{ '_height': 000 }" 형태로 적용합니다. (단위 px)
* 설정한 높이까지만 내용이 표시되고 세로 스크롤바가 노출됩니다.

# 좌측 고정
* th, td에 __table-fixed 클래스를 추가합니다.
* 건너뛰기 없이 0번 째 열부터 차례로 적용해야 합니다.

그리드 테이블_isGrid

# 그리드 테이블
* data-table 속성으로 '_isGrid': true 값을 추가합니다.

# 정렬
* th 클릭으로 내림차순/오름차순 기능이 적용됩니다. * th에 tabindex="0" 속성을 추가합니다. (정렬기능 필요없는 th에는 tabindex 속성 삭제)
* 정렬 시 th에 내림차순에는 __descending, 오름차순에는 __ascending 클래스가 추가됩니다.
* data-table 속성으로 onOrderBy 콜백을 연결할 수 있습니다.

# 순서변경
* th를 드래그앤드롭으로 순서를 변경할 수 있습니다. * data-table 속성으로 onColumnChange 콜백을 연결할 수 있습니다.

테이블 + 상하단mm_tablebox, mm_tablebox-caption, mm_tablebox-unit

# 기본
* mm_table 상하단에 추가되는 요소가 있을 경우 mm_tablebox로 감싸줍니다.
* mm_tablebox-unit은 상하단 동일하게 맞춰줍니다.
* 이미지가 들어간 테이블에는 상단에 안내문구를 넣어줍니다.

# 순서편집 기능
* mm_tablebox에 __table_sortable__ 클래스를 추가합니다.
* 리스트 내에 순서편집 기능을 제외할 tr에 __sortable_excepted__ 클래스를 추가하고,
  해당 tr 체크박스 영역에는 <span class="text_check-none">-</span> 를 추가합니다.
